Renewal

*Renewal Preprocess Is Required
*Renewal Validator Is Required
*Renewal Validator Is Required
When "Renewal count is part of product number" is enabled, do not forget to put two 0's (that is 00) as part of policy number suffix.

Renewal pre-processor should be a module.method that will process the uploaded renewable records.
Parameters:

  • row (renewable record with data field that has all data in json format)
  • product (product object with data)
  • pgroup (product group/LoB)
Ex: lib/pas.renewal_preprocessor

Renewal validator should be a module.method that will be called when the renew API is called (validation at the time of renewal start).
Parameters:

  • row (renewable record with data field that has all data in json format)
  • product (product object with data)
Ex: api/v1/pas._renewal_validator

*Required Field
*Quote No Version Is Required
*Quote Id Version Is Required

Version number pattern is used when the quotatoin is cloned. The grand-father quote_id will always be used as base quote id and a sequence number, one greater than most recent quote will be used as suffix. Even if the cloned quotation is cloned, top most quotation id will always be used as base quotation id (however the data will be from that of the cloned quote)

Version number patten should end with 0's as many as needed to support maximum number. It may be prefixed with a non numeric string as separator. Ex: "-00".

Please note, the separator for quote_id version must be a valid file name character as this will be used as temporary folder name when generating PDF files.
